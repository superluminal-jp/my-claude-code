# Data Model: DDD ドメインモデル管理スキル

**Date**: 2026-05-09

スキルが扱うすべてのデータ構造（ファイル・テーブル・キュー）を定義する。

---

## 1. セッション内候補キュー（永続化なし）

| フィールド | 型 | 説明 |
|---|---|---|
| `candidate_term` | string | 検出された概念名 |
| `source_text` | string | 検出元の引用文 |
| `trigger_type` | enum | `aggregate` / `entity` / `value-object` / `domain-event` / `invariant` |
| `detected_at` | int | 会話ターン番号 |
| `context` | string | 推定された Bounded Context 名（不明の場合は `"unknown"`） |

## 3. `docs/models/<context-kebab>.md`

### ヘッダー

|---|---|
| `Context` | コンテキスト表示名 |
| `Last Updated` | YYYY-MM-DD |

### 3-1. Mermaid classDiagram

テーブルセクション（3-2 〜 3-6）から機械生成。
- ステレオタイプ: `<<Aggregate Root>>` / `<<Entity>>` / `<<Value Object>>` / `<<Domain Event>>`
- 集約内包含: `*--`（コンポジション）
- 集約間参照: `-->`（関連）

乖離検出: テーブル変更後に図との整合性を確認し、乖離があれば再生成を提案する。

### 3-2. 集約テーブル（7 フィールド）

| フィールド | 必須 | 説明 |
|---|---|---|
| 集約名 | 必須 | 集約の名前（日本語可） |
| 集約ルート | 必須 | ルートエンティティのクラス名 |
| 責務 | 必須 | この集約が保護する整合性の説明 |
| 境界 | 必須 | 集約に含まれるエンティティ・VO の一覧 |
| 不変条件 | 必須 | 常に維持されるビジネスルール（複数行は `/` 区切り） |
| ライフサイクル | 必須 | 状態遷移（矢印記法: `未確定→確定済→完了`） |
| 実装名 | 任意 | クラス名・モジュール名（コード未存在の場合は空欄） |

### 3-3. エンティティテーブル（6 フィールド）

| フィールド | 必須 | 説明 |
|---|---|---|
| エンティティ名 | 必須 | エンティティの名前 |
| 識別子 | 必須 | ID フィールド名と型 |
| 属性 | 必須 | 主要属性（複数は `,` 区切り） |
| 振る舞い | 必須 | 主要メソッド・操作 |
| 所属集約 | 必須 | どの集約に属するか |
| 実装名 | 任意 | クラス名（コード未存在の場合は空欄） |

### 3-4. 値オブジェクトテーブル（6 フィールド）

| フィールド | 必須 | 説明 |
|---|---|---|
| VO 名 | 必須 | 値オブジェクトの名前 |
| 構成要素 | 必須 | 含まれるフィールド |
| 検証ルール | 必須 | 生成時の不変条件 |
| 不変性理由 | 必須 | なぜ変更不可か |
| 使用箇所 | 必須 | 参照する集約・エンティティ |
| 実装名 | 任意 | クラス名（コード未存在の場合は空欄） |

### 3-5. ドメインイベントテーブル（5 フィールド）

| フィールド | 必須 | 説明 |
|---|---|---|
| イベント名 | 必須 | 過去形の命名（UL が正）|
| トリガー | 必須 | どの操作がこのイベントを発生させるか |
| ペイロード | 必須 | イベントが運ぶデータ |
| 後続処理 | 必須 | このイベントを受け取るハンドラ・リアクション |
| 実装名 | 任意 | クラス名・イベント型名（コード未存在の場合は空欄） |

### 3-6. 不変条件リスト（3 フィールド）

| フィールド | 必須 | 説明 |
|---|---|---|
| 条件 | 必須 | ビジネスルールの自然言語記述 |
| 制約 | 必須 | 技術的な制約への翻訳 |
| 違反時の扱い | 必須 | 例外・拒否・補正の種別と説明 |
