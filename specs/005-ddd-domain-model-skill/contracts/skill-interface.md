# Skill Interface Contract: domain-model

**Date**: 2026-05-09

スキルが提供するコマンド・入力・出力・副作用の契約を定義する。

---

## 呼び出し方法

### 明示コマンド（ユーザーが能動的に起動）

| 呼び出し例 | 動作 |
|---|---|
| `「注文コンテキストのドメインモデルを作成して」` | Bootstrap フロー（既存ファイルなし）または Update フロー（既存あり） |
| `「ドメインモデルを更新して」` | 候補キューの内容を元に対象コンテキストの Update フロー |
| `「domain-model スキルを起動して」` | 上記と同様 |

### 受動起動（スキルが自動検出）

会話中に DDD 語彙パターンを検出した場合、候補キューに積む。提示条件は以下の両方を満たすとき:
1. キューに 1 件以上の未提示候補がある
2. 直前のターンで新規 DDD 語彙候補が検出されなかった

---

## 入力

| 入力 | 必須 | 説明 |
|---|---|---|
| コンテキスト名 | 任意 | 指定がなければ会話から推定または質問 |
| `docs/ubiquitous-language.md` | 任意 | 存在すればドメインイベント命名の正として読み込む |
| 既存の `docs/models/<context>.md` | 任意 | 存在すれば Update フロー（差分提示） |

---

## 出力

| 出力 | 説明 |
|---|---|
| `docs/models/index.md` | BC 一覧 + コンテキスト間関係図（初回作成または同期更新） |
| `docs/models/<context-kebab>.md` | BC 別モデルファイル（新規作成または更新） |

いずれも **差分提示 → 明示確認後に書き込み**。確認前はファイルを変更しない。

---

## 副作用（禁止事項）

- `docs/ubiquitous-language.md` への書き込みは行わない（読み取り専用）
- `docs/context-map.md` への書き込みは行わない（UL スキルが管理）
- 複数コンテキストのファイルを 1 回の確認でまとめて書き込まない（コンテキストごとに確認）

---

## エラー条件

| 条件 | 動作 |
|---|---|
| コンテキスト名が特定できない | 「どのコンテキストのモデルを操作しますか？」と質問 |
| 同一コンテキスト名で内容が衝突 | 「同名のコンテキストが既に存在します。上書きしますか？別名にしますか？」 |
| コンテキスト間概念衝突 | 分離か改名かをユーザーに選択させる。サイレントマージは禁止 |
| Mermaid 図とテーブルの乖離 | 「図がテーブルと一致しません。再生成しますか？」と提案 |
